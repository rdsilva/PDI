Atividade 3.2 A

  • Nesta atividade deve-se solicitar ao usuário uma imagem e carregá-la, em seguida deve-se perguntar para o usuário por duas coordenadas de uma região dentro da imagem e negativá-la. Para a minha solução o usuário deverá informar alguma imagem e então utilizar o próprio mouse para gerar a região desejada, bastando clicar e arrastar entre o ponto inicial desejado e o ponto final.

#include<opencv2/core/core.hpp>
#include<opencv2/highgui/highgui.hpp>
#include<opencv2/imgproc/imgproc.hpp>
#include<iostream>
#include<string>

int px_1,py_1, px_2, py_2;
bool clicado = false;
cv::Mat imgOriginal;		// imagem original
cv::Mat imgGrayscale;		// imagem em escala de cinza
cv::Mat imgResult;		// imagem em escala de cinza com regi„o negativa



///////////////////////////////////////////////////////////////////////////////////////////////////
/*
Este metodo reconhece as coordenadas informadas pelo click do mouse
Verifica quais s„o os pontos maximo e minimos para X e Y
Percorre a matriz partindo do ponto minimo ao maximo invertendo o valor do pixel

A invers„o do valor do pixel d·-se pelo complemento de 255, ou seja
valor_novo = 255 - valor_atual
*/
void Negativar() {

	std::cout << "\n-------- Negativando a ROI definida...\n";

	int max_X = int(fmax(px_1, px_2));
	int max_Y = int(fmax(py_1, py_2));
	int min_X = int(fmin(px_1, px_2));
	int min_Y = int(fmin(py_1, py_2));

	// criando a regi„o com negativo da imagem
	cv::cvtColor(imgOriginal, imgResult, CV_BGR2GRAY);

	for (int i = min_X; i<max_X; i++) {
		for (int j = min_Y; j<max_Y; j++) {
			imgResult.at<uchar>(j, i) = 255 - imgResult.at<uchar>(j, i);
		}
	}

	cv::imshow("imgResult", imgResult);

	std::cout << "-------- ROI Negativada...\n\n";

}

///////////////////////////////////////////////////////////////////////////////////////////////////
/*
O mÈtodo de CallBack trata o uso de eventos do mouse sobre uma imagem
Neste caso ele È respons·vel armazenar os pontos de clique do mouse e
por chamar o metodo Negativar, respons·vel por gerar o negativo da ROI
*/

void CallBackFunc(int event, int x, int y, int flags, void* userdata){
	// Primeiro ponto, quando o bot„o esquerdo do mouse È clicado
	if (event == cv::EVENT_LBUTTONDOWN && !clicado){
		px_1 = x;
		py_1 = y;
		clicado = true;
		std::cout << "Mouse clicou na posicao 1 - (" << x << ", " << y << ")" << std::endl;
	}
	// Segundo ponto, quando o bot„o esquerdo do mouse È liberado
	if (event == cv::EVENT_LBUTTONUP && clicado){
		px_2 = x;
		py_2 = y;
		clicado = false;
		std::cout << "Mouse arrastado ate a posicao 2 - (" << x << ", " << y << ")" << std::endl;

		Negativar();
	}
}

Atividade 3.2 B

  • Nesta atividade deve-se usar o mesmo principio de seleção de uma região da imagem, contudo, deve-se carregar uma imagem trocar as regiões desta de forma aleatória. A minha solução calcula 4 ROIs (Regions of Interest - Regiões de Interesse) da imagem, de forma automatica, e de forma randomizada calcula a ordem em que estas serão dispostas, formando uma nova imagem e seguindo a distribuição das áreas em Z.

#include<opencv2/core/core.hpp>
#include<opencv2/highgui/highgui.hpp>
#include<opencv2/imgproc/imgproc.hpp>
#include<iostream>
#include<algorithm>
#include<stdlib.h>
#include<time.h>

int px_1, py_1, px_2, py_2;
bool clicado = false;
cv::Mat imgOriginal;		// imagem original
cv::Mat imgGrayscale;		// imagem em escala de cinza
cv::Mat imgResult;			// imagem resultado em escala de cinza
cv::Mat listOfRoi[4];		// lista de ROIs criadas


void pickARoi() {
	int index[4] = { 0,1,2,3 };

	srand(time(0));
	std::random_shuffle(std::begin(index), std::end(index));

	// criando indices temporarios apenas para ficar "melhor legivel"
	int ind1 = index[0];
	int ind2 = index[1];
	int ind3 = index[2];
	int ind4 = index[3];

	// imprimindo apenas para checar a ordem
	std::cout << "\n\nOrdem : { " << ind1 << ", " << ind2 << ", " << ind3 << ", " << ind4 << " }\n\n";

	listOfRoi[ind1].copyTo(imgResult(cv::Rect(0,					0,							listOfRoi[ind1].cols,	listOfRoi[ind1].rows)));	// regi„o 1 em formato Z
	listOfRoi[ind2].copyTo(imgResult(cv::Rect(listOfRoi[ind1].cols, 0,							listOfRoi[ind2].cols,	listOfRoi[ind2].rows)));	// regi„o 2
	listOfRoi[ind3].copyTo(imgResult(cv::Rect(0,					listOfRoi[ind1].rows,		listOfRoi[ind3].cols,	listOfRoi[ind3].rows)));	// regi„o 3
	listOfRoi[ind4].copyTo(imgResult(cv::Rect(listOfRoi[ind1].cols, listOfRoi[ind1].rows,		listOfRoi[ind4].cols,	listOfRoi[ind4].rows)));	// regi„o 4

}

Atividade 4.2

  • Esta atividade pede para que o aluno desenvolva uma solução de labeling para qualquer imagem mas que rompa a restrição dada no exemplo de apenas 255 labels. Fazendo uma breve analise, realmente na forma proposta só podemos utilizar apenas 255 labels devido ao formato em que a imagem e lida na memória, utilizando apenas 8 bits. Temos hoje em nossas mãos celulares que utilizam processadores de 64 bits, ou seja, podemos facilmente utilizar 16 bits, o que nos daria um total de 65536 opções de labels.

Atividade 5.2 A

  • Esta atividade já é um pouco mais evoluída que as demais, forçando o aluno a trabalhar agora com vídeo. Apesar de um vídeo não ser nada mais que uma sequência de imagens, o processamento em tempo real é extremamente sensível. Para esta atividade o aluno deverá equalizar em tempo real a imagem que é capturada. Para a minha solução eu capturo o quadro do vídeo (uma imagem), converto para YCbCr, aplico a equalização apenas no canal Y e reconverto a imagem para RGB. A grande sacada é que equalizar uma imagem nada mais é que equalizar os níveis de intensidade luminosa desta, ou seja, como o canal Y é o canal da intensidade luminosa, equalizando este eu equalizo a imagem.

#include <iostream>
#include <opencv2\opencv.hpp>
#include <opencv2\highgui\highgui.hpp>
#include <opencv2\imgproc\imgproc.hpp>

using namespace cv;
using namespace std;

int width, height;
int nbins = 64;
int histw = nbins, histh = nbins / 2;
float range[] = { 0, 256 };
const float *histrange = { range };
bool uniform = true;
bool acummulate = false;

Mat histograma(Mat image, vector<Mat> planes, Mat histImgR, Mat histImgG, Mat histImgB, Mat histR, Mat histG, Mat histB) {
	// calculando o histograma da imagem original
	calcHist(&planes[0], 1, 0, Mat(), histR, 1,
		&nbins, &histrange,
		uniform, acummulate);
	calcHist(&planes[1], 1, 0, Mat(), histG, 1,
		&nbins, &histrange,
		uniform, acummulate);
	calcHist(&planes[2], 1, 0, Mat(), histB, 1,
		&nbins, &histrange,
		uniform, acummulate);

	// normalizando o histograma
	normalize(histR, histR, 0, histImgR.rows, NORM_MINMAX, -1, Mat());
	normalize(histG, histG, 0, histImgG.rows, NORM_MINMAX, -1, Mat());
	normalize(histB, histB, 0, histImgB.rows, NORM_MINMAX, -1, Mat());

	histImgR.setTo(Scalar(0));
	histImgG.setTo(Scalar(0));
	histImgB.setTo(Scalar(0));

	// linearizando o historgama
	for (int i = 0; i<nbins; i++) {
		line(histImgR, Point(i, histh),
			Point(i, cvRound(histR.at<float>(i))),
			Scalar(0, 0, 255), 1, 8, 0);
		line(histImgG, Point(i, histh),
			Point(i, cvRound(histG.at<float>(i))),
			Scalar(0, 255, 0), 1, 8, 0);
		line(histImgB, Point(i, histh),
			Point(i, cvRound(histB.at<float>(i))),
			Scalar(255, 0, 0), 1, 8, 0);
	}

	// apresentando o histograma
	histImgR.copyTo(image(Rect(0, 0, nbins, histh)));
	histImgG.copyTo(image(Rect(0, histh, nbins, histh)));
	histImgB.copyTo(image(Rect(0, 2 * histh, nbins, histh)));

	return image;
}

int main(int argc, char** argv) {
	Mat imageOriginal, imageResult;
	Mat histR, histG, histB;
	Mat histREq, histGEq, histBEq;
	VideoCapture cap;
	vector<Mat> planes;
	vector<Mat> planesEq;

	cap.open(1);

	if (!cap.isOpened()) {
		cout << "cameras indisponiveis\n";
		system("pause");
		return -1;
	}

	width = cap.get(CV_CAP_PROP_FRAME_WIDTH);
	height = cap.get(CV_CAP_PROP_FRAME_HEIGHT);

	cout << "largura = " << width << endl;
	cout << "altura  = " << height << endl;

	Mat histImgR(histh, histw, CV_8UC3, Scalar(0, 0, 0));
	Mat histImgG(histh, histw, CV_8UC3, Scalar(0, 0, 0));
	Mat histImgB(histh, histw, CV_8UC3, Scalar(0, 0, 0));

	Mat histImgREq(histh, histw, CV_8UC3, Scalar(0, 0, 0));
	Mat histImgGEq(histh, histw, CV_8UC3, Scalar(0, 0, 0));
	Mat histImgBEq(histh, histw, CV_8UC3, Scalar(0, 0, 0));

	while (1) {
		// capturando a imagem
		cap >> imageOriginal;

		// separando os canais da imagem original
		split(imageOriginal, planes);


		// EQUALIZANDO A IMAGEM
		// ---------------------------------------------------------------------------

		// copiando e convertendo a imagem original para YCrCb
		cvtColor(imageOriginal, imageResult, CV_BGR2YCrCb);

		// separando os canais da imagem convertida
		split(imageResult, planesEq);

		// equalizando o canal Y (intensidade) da imagem convertida
		equalizeHist(planesEq[0], planesEq[0]);

		// agrupando os canais da imagem convertida
		merge(planesEq, imageResult);

		// convertendo a imagem manipulada para o dominio de cores original
		cvtColor(imageResult, imageResult, CV_YCrCb2BGR);

		// ---------------------------------------------------------------------------

		// CALCULAR E APRESENTAR HISTOGRAMAS
		// ---------------------------------------------------------------------------

		imageOriginal = histograma(imageOriginal, planes, histImgR, histImgG, histImgB, histR, histG, histB);
		imageResult = histograma(imageResult, planesEq, histImgREq, histImgGEq, histImgBEq, histREq, histGEq, histBEq);

		// ---------------------------------------------------------------------------

		// apresentando as imagens
		imshow("Original", imageOriginal);
		imshow("Resultado", imageResult);
		if (waitKey(30) >= 0) break;
	}
	return 0;
}

Atividade 5.2 B

  • Esta atividade é um desafio um pouco maior que a atividade anterior. Deve-se monitorar uma imagem e detectar movimento nesta quando ocorrer. Utilizando basicamente o mesmo principio da atividade anterior, converto a imagem para o modelo HSV, crio uma ROI que será utilizada para monitorar os canais H (hue - valor) e S (saturation - saturação), calculo o histograma para a imagem capturada e para a ROI, utilizo o algoritmo nativo do OpenCV Bhattacharyya para calcular a correlação entre os histogramas, e calculei empiricamente um threshold para detectar um movimento suave. Para o threshold quanto menor o valor resultado mais sensível ao movimento será este algoritmo.

#include <iostream>
#include <opencv2/opencv.hpp>

using namespace cv;
using namespace std;

int main(int argc, char** argv) {
	Mat image, imageHSV;
	int width, height;
	int count = 0;
	double last_value = 0.0;
	double threshold = 1.6;
	VideoCapture cap;

	cap.open(0);

	if (!cap.isOpened()) {
		cout << "cameras indisponiveis";
		return -1;
	}

	width = cap.get(CV_CAP_PROP_FRAME_WIDTH);
	height = cap.get(CV_CAP_PROP_FRAME_HEIGHT);

	cout << "largura = " << width << endl;
	cout << "altura  = " << height << endl;

	while (1) {
		cap >> image;

		cvtColor(image, imageHSV, COLOR_BGR2HSV);

                // criando uma ROI para comparação
		Mat hsv_half_down = imageHSV(Range(imageHSV.rows / 2, imageHSV.rows - 1), Range(0, imageHSV.cols - 1));

		// usando 50 pontos para o canal H e 60 para o S
		int h_bins = 50; int s_bins = 60;
		int histSize[] = { h_bins, s_bins };

		// range de variação para H e S
		float h_ranges[] = { 0, 180 };
		float s_ranges[] = { 0, 256 };

		const float* ranges[] = { h_ranges, s_ranges };

		// usando apenas os canais H e S
		int channels[] = { 0, 1 };

		// histogramas
		MatND hist_base;
		MatND hist_half_down;

		// calculando os histogramas da imagem original e da ROI
		calcHist(&imageHSV, 1, channels, Mat(), hist_base, 2, histSize, ranges, true, false);
		normalize(hist_base, hist_base, 0, 1, NORM_MINMAX, -1, Mat());

		calcHist(&hsv_half_down, 1, channels, Mat(), hist_half_down, 2, histSize, ranges, true, false);
		normalize(hist_half_down, hist_half_down, 0, 1, NORM_MINMAX, -1, Mat());

		// fazendo a comparação enter os histogramas

		int compare_method = 4;
		double base_half = compareHist(hist_base, hist_half_down, compare_method);

		double test = abs(base_half - last_value);

		if (test > threshold) {
			printf("%3d --> MOVIMENTO \n", count);
			last_value = base_half;
			count++;
		}

		imshow("image", image);
		if (waitKey(30) >= 0) break;
	}
	return 0;
}
